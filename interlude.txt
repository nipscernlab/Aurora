O DIRETOR (diretor.json LOCALIZADO EM componentes/Scripts/diretor.json) É RESPONSÁVEL POR ARMAZENAR OS ESTADOS DA INTERFACE: MODO PROCESSOR, MODO PROJECT & SIMULATION, MODO PROJECT,
FILE TREE, HIERARQUICAL TREE, VERILOG TREE, BOTÕES DISABLED/ENABLED, PROJETO ATUAL. diretor.js CONTROLA ESTE ARQUIVO E OS ESTADOS

ABRA A INTERFACE AURORA: VERIFIQUE SE A INTERFACE FOI FECHADA COM UM PROJETO ABERTO, SE SIM, ABRA O PROJETO,
CONFIGURE O MODO DE COMPILAÇÃO DA MANEIRA QUE FOI CONFIGURADO NA ÚLTIMA SEÇÃO

A ÁRVORE DE ARQUIVOS DEVE SALVAR QUAL ÁRVORE FOI USADA NA ÚLTIMA SEÇÃO E TENTAR CARREGÁ-LA, CASO NÃO CONSIGA
EXIBA FILE TREE

APÓS CARREGAR O PROJETO E AS CONFIGURAÇÕES PRÉVIAS DO USUÁRIO, AGORA O USUÁRIO TEM A LIBERDADE DE CONSTRUIR
SEU PROJETO

MODO PROCESSOR: VIA PROCESSOR HUB O USUÁRIO CRIA PROCESSADORES, CONFIGURA VIA PROCESSOR SETTINGS, COMPILA
PASSO A PASSO(C+-, ASM, VERILOG, GTKWAVE, PRISM) OU VIA FULL BUILD

MODO PROJECT COM SIMULAÇÃO: CONFIGURA A COMPILAÇÃO DO PROJETO NO SETTINGS PROJECT MODE, COMPILA (C+-, ASM, VERILOG, GTKWAVE, PRISM) OU VIA FULL BUILD.

MODO PROJECT SEM SIMULAÇÃO: VERIFICAÇÃO DOS ARQUIVOS VERILOG CRIADOS, VISUALIZAÇÃO COM GTKWAVE, VISUALIZAÇÃO COM PRISM

SE ESTIVERMOS NO MODO PROJECT SEM SIMULAÇÃO, A ÁRVORE DEVE SER TROCADA PARA ÁRVORE VERILOG MODE, ONDE
EXIBIMOS SOMENTE OS ARQUIVOS VERILOG IMPORTADOS PARA O PROJETO. CADA ARQUIVO IMPORTADO OU CRIADO NA ÁRVORE
DEVE TER UM BOTÃO DE DOIS ESTADOS ASSOCIADO. SE O BOTÃO DE DOIS ESTADOS ESTIVER LIGADO PARA A ESQUERDA, SIGNIFICA QUE O ARQUIVO É SINTETIZÁVEL, SE O BOTÃO ESTIVER LIGADO A ESQUERDA, SIGNIFICA QUE O ARQUIVO É TESTBENCH. SOMENTE UM ARQUIVO PODE SER SET-AS-TOPLEVEL E SOMENTE UM ARQUIVO PODE SER SET-AS-TESTBENCH. TODOS
OS OUTROS ARQUIVOS DO PROJETO DEVEM IR PARA A COMPILAÇÃO

O USUÁRIO, NO MESMO MENU DE SET-AS-TOPLEVEL/SET-AS-TESTBENCH, PODE SETAR PARA O ARQUIVO SER EXCLUÍDO DA COMPILAÇÃO, MAS PERMANECER NO PROJETO

OS .V DE TODOS OS PROCESSADORES CRIADOS DEVEM SER AUTOMATICAMENTE USADOS NA COMPILAÇÃO
